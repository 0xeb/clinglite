# Building clinglite

## Prerequisites

- **CMake** >= 3.20
- **C++17 compiler**: GCC 11+, Clang 15+, or MSVC 2022
- **Python 3** (for patch application and build scripts)
- **Ninja** (recommended) or Make
- **Disk space**: ~15-30 GB (LLVM/Clang/Cling sources + build artifacts)
- **Build time**: 30-90 minutes for the LLVM/Cling build (depends on CPU cores and disk speed)

## End-to-end (from source)

```bash
# 1. Clone with submodules (shallow — only fetches pinned commits, not full LLVM history)
git clone --recurse-submodules --shallow-submodules https://github.com/0xeb/clinglite.git
cd clinglite

# 2. Build Cling/LLVM (one-time, ~30-90 min)
#    Patches are applied automatically during CMake configure.
python scripts/build_cling.py -j $(nproc)   # Linux / macOS
python scripts/build_cling.py               # Windows (auto-detects VS)

# 3. Build clinglite
python scripts/build_clinglite.py

# 4. Run an example
./build-ninja-static/examples/basic            # Linux / macOS
.\build-vs-static\examples\Release\basic.exe   # Windows
```

> **Tip:** The `--shallow-submodules` flag fetches only the pinned commit for `llvm-project` and `cling-src`, saving ~3 GB of git history. If you already cloned without it, you can re-init with:
> ```bash
> git submodule update --init --depth 1
> ```

## Build scripts

**`scripts/build_cling.py`** — builds the Cling/LLVM tree:

```bash
python scripts/build_cling.py                # full build
python scripts/build_cling.py --configure-only  # CMake configure only
python scripts/build_cling.py --build-only      # skip configure, just build
python scripts/build_cling.py -j 8              # limit parallel jobs
python scripts/build_cling.py --ninja           # use Ninja on Windows (requires Developer Command Prompt)
```

Output: `build-cling/` directory with the LLVM/Cling build tree.

**`scripts/build_clinglite.py`** — builds the clinglite library:

```bash
python scripts/build_clinglite.py             # static (VS on Windows, Ninja on Linux/macOS)
python scripts/build_clinglite.py --shared    # shared library variant
python scripts/build_clinglite.py --fat       # fat-static (Linux/macOS, all LLVM/Cling merged into one .a)
python scripts/build_clinglite.py --ninja     # force Ninja on Windows
```

## CMake presets

```bash
# Windows (generates .sln)
cmake --preset vs-static
cmake --build build-vs-static --config Release

# Linux / macOS
cmake --preset ninja-static
cmake --build build-ninja-static
```

Available presets (`CLING_BUILD_DIR` defaults to `build-cling/`):

| Preset | Description |
|--------|-------------|
| `vs-static` | VS 2022 — static library (multi-config) |
| `vs-shared` | VS 2022 — shared DLL (multi-config) |
| `ninja-static` | Ninja — static library (Release) |
| `ninja-shared` | Ninja — shared library (RelWithDebInfo) |
| `fat-static` | Fat static archive — all LLVM/Cling merged into one `.a` (Linux/macOS only) |

## Manual build

```bash
mkdir build && cd build
cmake .. -DCLING_BUILD_DIR=/path/to/cling-build
cmake --build . -j$(nproc)
```

## Build type

By default, clinglite builds as a **static library** (`libclinglite.a` / `clinglite.lib`). Override with:

```bash
cmake .. -DCLING_BUILD_DIR=... -DCLINGLITE_BUILD_SHARED=ON   # force shared
cmake .. -DCLING_BUILD_DIR=... -DCLINGLITE_BUILD_SHARED=OFF  # force static
```

The `CLINGLITE_API` export macro is auto-generated by CMake's `GenerateExportHeader` and handles `__declspec(dllexport/dllimport)` on Windows automatically.

## add_subdirectory (supported integration)

If clinglite lives as a subdirectory (monorepo or git submodule):

```cmake
add_subdirectory(clinglite)
target_link_libraries(myapp PRIVATE clinglite::clinglite)
```

`find_package()` / install-prefix consumption is intentionally not part of the current public contract. Build and consume clinglite from source on the same machine.
